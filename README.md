# 우동 실거래 일일 알림

매일 아침 7시(KST)에 해운대구 우동의 신규 실거래를 텔레그램으로 알려주는 자동화.

## 동작 흐름

```
매일 07:00 KST  GitHub Actions 자동 실행
     ↓
국토부 API에서 이번 달 + 지난 달 거래 조회
     ↓
'우동' 거래만 필터
     ↓
이전에 본 거래 ID와 비교 → 신규 / 새 취소만 추출
     ↓
텔레그램 봇으로 메시지 전송
  (마린시티 단지는 ⭐ 강조 / 전고점 돌파 시 🎯 표시)
     ↓
본 거래 ID들을 cache/seen_ids.json에 저장
```

## 설정 방법

### 1. GitHub Secrets 등록

저장소의 **Settings → Secrets and variables → Actions → New repository secret**에 3개 등록:

| 이름 | 값 |
|------|-----|
| `SERVICE_KEY` | 국토부 실거래가 API 키 |
| `TELEGRAM_BOT_TOKEN` | BotFather에서 받은 봇 토큰 |
| `TELEGRAM_CHAT_ID` | 본인 텔레그램 Chat ID |

### 2. 수동 실행으로 테스트

저장소의 **Actions** 탭 → **Daily Udong Alert** → **Run workflow** 클릭

성공하면 텔레그램으로 메시지가 도착합니다.

### 3. 자동 실행

수동 실행이 잘 되면 끝. 매일 07:00 KST에 자동으로 실행됩니다.

## 커스터마이징

`daily_check.py` 상단 설정:

- `TARGET_DONG = '우동'` — 다른 동으로 변경 가능 (좌동, 중동 등)
- `MARINE_CITY_KEYWORDS` — ⭐ 강조 단지 목록
- 시간 변경: `.github/workflows/daily.yml`의 `cron: '0 22 * * *'`
  - KST 07:00 = UTC 22:00 (전날)
  - KST 08:00을 원하면 `'0 23 * * *'`

## 주의사항

- **국토부 신고는 계약일로부터 30일 이내**가 의무 → 계약 직후 즉시 보이지는 않음
- 거래가 많지 않은 날은 0건일 수 있음 (정상)
- 같은 거래가 한 번만 알림 (이미 본 거래는 캐시에 저장돼 재알림 안 함)
